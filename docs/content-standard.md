# 콘텐츠 작성 기준

이 프로젝트의 글은 초보자에게는 쉽게 읽히고, 포트폴리오를 보는 사람에게는 기술 이해의 깊이가 보이도록 작성합니다.

## 기본 구조

모든 교재 글은 다음 순서를 따릅니다.

1. 한 줄 결론
2. 쉬운 비유
3. 실제 구조
4. 현업 키워드
5. 공식 자료 기반 근거
6. 체크 질문

## 레벨별 깊이

- 기초: 용어와 큰 그림을 먼저 설명합니다. 수식과 제품 세대보다 “왜 필요한가”를 앞에 둡니다.
- 중급: 실제 구조와 트레이드오프를 설명합니다. 표, 비교, 현업 키워드를 사용합니다.
- 심화: 수율, 양산성, 열, 전력, 패키징, 고객 검증처럼 실무 의사결정 기준까지 다룹니다.

한 글 안에서도 기초 설명에서 시작해 심화 관점으로 올라가야 합니다. 쉬운 설명만 남기거나, 어려운 용어만 나열하는 방식은 피합니다.

## 복습 질문 원칙

- 질문은 단순 정의보다 “왜 필요한가”와 “무엇을 trade-off 하는가”를 묻습니다.
- 답변에는 쉬운 핵심 답과 실무 포인트를 분리해 제공합니다.
- 실무 포인트에는 성능, 전력, 비용, 수율, 패키징, 고객 검증 중 최소 하나를 포함합니다.
- 사용자가 틀린 질문에서 바로 관련 글로 돌아갈 수 있어야 합니다.

## 문장 원칙

- 한 문단에는 하나의 판단만 담습니다.
- 낯선 용어는 영어 원어를 함께 씁니다.
- 단순 비유로 시작하되, 반드시 실제 구조로 돌아옵니다.
- 회사 공식 자료의 문장을 그대로 옮기지 않습니다.
- 수치와 제품 세대는 출처를 확인한 뒤 작성합니다.
- 최신 제품명, 세대, 양산 일정처럼 바뀔 수 있는 정보는 `updatedAt`을 갱신하고 본문에 확인 근거를 남깁니다.

## 출처 사용 방식

- `sources`에는 공식 자료를 우선 연결합니다.
- 본문에서는 출처 문장을 복사하지 않고 “공식 표현 → 기술 의미 → 학습자 질문” 순서로 재구성합니다.
- 각 글 하단에는 어떤 출처가 어떤 설명의 근거인지 짧게 적습니다.
- 출처가 제품 홍보 문구일 때는 성능 주장과 엔지니어링 제약을 분리해 설명합니다.
