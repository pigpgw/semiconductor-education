# 보완 기획

이 문서는 1차 MVP 이후 무엇을 먼저 보완할지 정리합니다. 목표는 기능을 많이 붙이는 것이 아니라, 하나의 오픈소스 반도체 문서로서 신뢰와 깊이를 먼저 높이는 것입니다.

구체적인 정보 구조, 글 상세 UX, 레벨 UX, 출처/뉴스 보완 기준은 [설계 보완](design-supplement.md)을 따릅니다.

## 보완 방향

1. 콘텐츠 깊이를 먼저 올립니다.
2. 로그인 없는 접근성을 유지합니다.
3. 공식 출처 기반 설명을 강화합니다.
4. 포트폴리오에서 기획, 콘텐츠, 구현의 연결이 보이게 합니다.
5. 사용자에게 “내 수준에서 어디를 읽으면 되는지”를 더 분명히 알려 줍니다.
6. 기여자가 설명 개선, 출처 보강, 용어 추가처럼 작은 단위로 참여할 수 있게 합니다.

## P0 상세 기획

### 1. 핵심 글 3편 리라이트

1차 리라이트는 완료했습니다. DRAM은 스펙 읽기와 면적·전력·대역폭 trade-off, HBM은 적층 수·대역폭·패키징·고객 검증, EUV는 적용 레이어·overlay·결함·수율 판단 기준을 보강했습니다.

다음 리라이트는 문장 밀도를 더 낮추고, 도식과 본문 표를 더 강하게 연결하는 방향으로 진행합니다.

각 글은 다음 질문에 답해야 합니다.

- 기초 독자가 첫 문단에서 주제를 이해하는가?
- 중급 독자가 구조와 트레이드오프를 확인하는가?
- 심화 독자가 수율, 전력, 열, 양산성, 고객 검증 중 최소 두 가지를 이해하는가?
- 공식 자료의 표현을 그대로 베끼지 않고 기술 의미로 바꿨는가?
- 마지막 체크 질문이 정의 암기가 아니라 설명 능력을 요구하는가?

### 2. 출처 확인일 정리

각 MDX 글의 `sources`에는 공식 자료 링크를 유지하고, 본문 하단에는 다음 관점을 명시합니다.

- 이 출처가 어떤 주장의 근거인지
- 제품 홍보 문구와 엔지니어링 의미를 어떻게 분리했는지
- 최신 제품명이나 세대 정보가 바뀔 수 있는지

### 3. 포트폴리오 설명 보강

README와 문서에는 단순히 “반도체를 공부하는 사이트”가 아니라 다음 능력이 보이게 써야 합니다.

- 어려운 기술을 난이도별로 재구성하는 능력
- 공식 자료를 학습 콘텐츠로 바꾸는 능력
- 접근성, 무로그인, 모바일 가독성을 고려한 제품 설계 능력
- Git workflow, CI, 검증, 배포 파이프라인을 운영하는 능력

## 콘텐츠 확장 순서

| 순서 | 주제 | 먼저 다룰 이유 |
| --- | --- | --- |
| 1 | NAND와 SSD | DRAM과 비교하면 메모리 분류가 선명해짐 |
| 2 | DDR, LPDDR, GDDR, HBM 비교 | 제품군 차이가 취업/면접 질문으로 자주 연결됨 |
| 3 | 패키징, TSV, MR-MUF | HBM을 실무 수준으로 이해하는 데 필요함 |
| 4 | 8대 공정 입문 | EUV를 전체 공정 흐름 속에 배치할 수 있음 |
| 5 | 수율과 품질 | 양산성과 고객 검증 관점을 강화함 |
| 6 | 직무별 학습 경로 | 포트폴리오 평가자가 프로젝트 의도를 빠르게 이해함 |

## UX 보완 순서

| 순서 | 보완 | 기준 |
| --- | --- | --- |
| 1 | 글 상세 상단 요약 강화 | 긴 글을 읽기 전 핵심을 빠르게 파악 |
| 2 | 글 상세 관련 용어 탐색 | 본문을 읽다가 막히는 용어를 같은 화면에서 먼저 해소 |
| 3 | 복습 질문 필터 고도화 | 레벨 진단 결과와 복습 질문 추천 연결 |
| 4 | 학습 노트 내보내기 | 로그인 없이 학습 결과를 보관 |
| 5 | 출처 확인 배지 | 공식 자료 기반이라는 신뢰 강화 |
| 6 | 외부 링크 허브 | 기업 기술블로그를 쉽게 찾고 원문으로 이동 |

## 하지 않을 것

보완 단계에서도 다음은 우선순위가 낮습니다.

- 로그인 기반 개인화
- 커뮤니티 기능
- 조회수/좋아요 같은 참여 지표
- 광고성 랜딩 페이지
- 공식 출처 확인 없이 최신 제품 정보를 빠르게 올리는 방식

## 의사결정 기준

새 작업은 아래 질문을 통과할 때만 진행합니다.

1. 초보자에게 진입 장벽을 낮추는가?
2. 중급자에게 구조와 비교를 제공하는가?
3. 심화 학습자에게 실무 제약을 보여 주는가?
4. Samsung/SK 공식 자료를 더 잘 읽게 만드는가?
5. 포트폴리오에서 기획, 콘텐츠, 구현 역량이 더 선명해지는가?
