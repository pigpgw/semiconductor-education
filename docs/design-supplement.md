# 설계 보완

이 문서는 1차 MVP 이후 설계를 더 선명하게 만들기 위한 보완안입니다. 기준 문서는 [최종 설계 확정](final-design.md)이고, 이 문서는 다음 구현과 문서 리라이트를 어떤 순서로 진행할지 정리합니다.

## 보완 목표

Semiconductor Education은 기능이 많은 교육 서비스가 아니라, 반도체를 이해하는 방식을 정리한 오픈소스 문서입니다. 따라서 설계 보완의 목표는 다음 세 가지입니다.

1. 처음 들어온 사용자가 1분 안에 자기 시작점을 고른다.
2. 한 글 안에서 기초 설명부터 실무 판단까지 자연스럽게 내려간다.
3. 공식 기술블로그와 뉴스는 복제하지 않고 원문 이동과 학습 해설로 연결한다.

## 현재 설계의 부족한 점

| 영역 | 현재 상태 | 보완 필요 |
| --- | --- | --- |
| 첫 방문 흐름 | 홈, 레벨, 로드맵, 글 목록이 분리되어 있음 | 사용자가 `내 레벨 찾기`, `로드맵 보기`, `바로 읽기` 중 무엇을 고를지 더 명확해야 함 |
| 글 상세 | MDX 본문, 90초 요약, 레벨별 읽기 안내, 핵심 구조 다이어그램, 관련 용어, 공식 자료 해석 블록 구현 | 문장 밀도 조절과 복습 흐름 연결이 남음 |
| 레벨 진단 | 3단계 진단이 있음 | 진단 결과가 글 상세의 읽기 모드와 더 강하게 연결되어야 함 |
| 출처 활용 | 공식 출처 정책과 링크 전략이 있음 | 글 안에서 “이 출처가 어떤 주장에 쓰였는지”를 더 분명히 보여야 함 |
| 뉴스/기술블로그 | 링크 허브, 수동 큐레이션 목록, 상세 해설 노트 구현 | 자동 수집 전 공식 RSS/API 후보와 품질 기준을 더 정리해야 함 |
| 오픈소스 운영 | 전략은 있음 | 기여자가 작은 단위로 고칠 수 있는 문서 템플릿과 체크리스트가 필요함 |

## 정보 구조 보완

사이트는 다음 흐름으로 읽혀야 합니다.

```txt
홈
-> 레벨 진단
-> 로드맵
-> 글 목록
-> 글 상세
-> 용어 사전 / 복습 질문 / 공식 출처
```

MVP 이후에는 다음 두 경로를 추가합니다.

```txt
공식 출처 허브
-> 회사/분야별 기술블로그 링크
-> 관련 교재 글

산업 업데이트
-> 공식 발표 메타데이터
-> 사람이 검토한 학습 노트
-> 관련 교재 글
```

핵심은 사용자를 외부 뉴스에 오래 붙잡는 것이 아닙니다. 공식 자료를 본 뒤 다시 교재 글로 돌아와 개념을 정리하게 만드는 것입니다.

## 홈 보완 기준

홈은 예쁘게 보이는 랜딩 페이지가 아니라 학습 시작점입니다.

유지할 요소:

- 다크 문서형 첫 화면
- 칩 로고와 기술 그리드 배경
- `내 레벨 찾기`, `교재 읽기`, `GitHub` 진입
- 5개 학습 트랙
- 공식 출처 태그

보완할 요소:

- 첫 화면에서 “이 사이트는 뉴스 사이트가 아니라 오픈소스 문서”라는 정체성을 더 짧게 보여준다.
- `레벨을 모르면 여기부터`, `바로 읽고 싶으면 여기부터`, `전체 지도를 보고 싶으면 여기부터`를 명확히 분리한다.
- 핵심 글 3편은 단순 목록이 아니라 `왜 먼저 읽는지`를 함께 보여준다.
- 외부 출처 태그는 장식이 아니라 `/sources`로 이어지는 실제 링크가 되어야 한다.

## 글 상세 보완 기준

핵심 글은 길어질수록 더 읽기 쉬워야 합니다. 글 상세는 다음 순서로 보완합니다.

1. 90초 요약
2. 레벨별 읽기 안내
3. 본문 목차
4. 핵심 구조 다이어그램
5. 본문
6. 공식 출처와 확인일
7. 관련 용어
8. 체크 질문
9. 다음 글

### 90초 요약

긴 글 상단에는 다음 정보를 카드 하나로 제공합니다.

| 항목 | 내용 |
| --- | --- |
| 한 줄 결론 | 이 글이 답하는 핵심 질문 |
| 쉬운 비유 | 초보자가 처음 90초 안에 감각적으로 잡을 설명 |
| 예상 소요 시간 | 기초, 중급, 실무 관점별 읽기 시간 |
| 레벨별 읽기 목표 | 같은 글을 각 수준에서 어디까지 읽으면 되는지 |
| 먼저 알아야 할 말 | 최대 3개 용어 |
| 현업 키워드 | 수율, 열, 대역폭, 패키징 같은 실무 연결점 |
| 공식 출처 | Samsung, SK hynix 등 원문 링크 |
| 읽고 나면 | 사용자가 말할 수 있어야 하는 결과 |

### 레벨별 읽기 안내

레벨은 콘텐츠 잠금이 아니라 읽기 모드입니다.

| 레벨 | 글에서 먼저 볼 것 | 건너뛰지 말 것 |
| --- | --- | --- |
| 기초 | 한 줄 결론, 비유, 그림 | 체크 질문 1개 |
| 중급 | 구조, 비교표, 용어 | 트레이드오프 |
| 심화 | 공식 자료, 실무 키워드 | 양산성, 수율, 고객 요구 |

### 핵심 구조 다이어그램

핵심 구조 다이어그램은 장식이 아니라 본문 표를 읽기 위한 압축 지도입니다.

| 글 | 다이어그램이 보여 줄 판단 기준 |
| --- | --- |
| DRAM | 1T1C 셀, refresh, sensing, I/O width의 trade-off |
| HBM | DRAM die 적층, TSV, base die, interposer/package, 열과 수율 |
| EUV | DUV multi-patterning과 EUV selected layers, overlay, defect, throughput |

### 관련 용어

관련 용어는 별도 암기장이 아니라 본문을 다시 읽기 위한 보조 장치입니다.

| 항목 | 기준 |
| --- | --- |
| 선정 방식 | 글의 핵심 용어, 현업 키워드, 태그를 기준으로 관련 용어를 자동 연결 |
| 표시 내용 | 난이도, 카테고리, 영어 원어, 쉬운 설명, 실무 맥락 |
| 이동 방식 | 용어 제목은 `/glossary#용어명`으로 연결 |
| 제외 기준 | 본문 이해와 직접 관련 없는 추천 용어는 표시하지 않음 |

## 레벨 UX 보완

레벨 진단은 시험처럼 보이면 안 됩니다. 사용자가 자기 위치를 편하게 고르는 도구여야 합니다.

보완 원칙:

- 진단 질문은 정답 맞히기가 아니라 읽기 모드 추천으로 표현한다.
- 결과 화면에서 `추천 레벨`, `읽는 방법`, `바로 갈 글`을 함께 보여준다.
- 사용자가 언제든 직접 레벨을 바꿀 수 있어야 한다.
- 선택한 레벨은 서버가 아니라 브라우저에만 저장한다.
- 글 목록과 글 상세는 선택 레벨을 반영하되 다른 레벨 접근을 막지 않는다.

## 콘텐츠 보완 기준

각 핵심 글은 같은 주제를 세 층으로 설명합니다.

| 층 | 목적 | 예시 |
| --- | --- | --- |
| 개념 | 처음 듣는 사람이 큰 그림을 잡음 | DRAM은 작업대처럼 빠르게 쓰는 메모리 |
| 구조 | 실제 기술 요소를 연결함 | 셀, capacitor, refresh, I/O, bandwidth |
| 판단 | 실무 제약을 이해함 | 전력, 수율, 패키징, 고객 검증, 생산성 |

문장 기준:

- 한 문단에는 하나의 주장만 둔다.
- 어려운 용어는 처음 나올 때 괄호로 영어를 병기한다.
- 공식 자료의 문장을 복사하지 않고 기술 의미로 다시 쓴다.
- 제품 홍보 문구와 엔지니어링 의미를 분리한다.
- 모든 글은 마지막에 “자기 말로 설명하는 질문”을 둔다.

## 공식 출처와 뉴스 보완 기준

`/sources`와 `/industry`는 뉴스 사이트가 아니라 학습 보조 장치입니다.

### `/sources`

먼저 구현할 페이지입니다.

- 회사/기관 카드
- 분야 태그
- 대표 키워드
- 공식 사이트로 이동 버튼
- 이 출처를 읽을 때 봐야 할 관점

카드 클릭은 원문으로 이동합니다. 사이트 안에는 원문 본문을 저장하지 않습니다.

### `/industry`

1차 구현은 수동 큐레이션 목록으로 시작합니다.

- 공식 RSS/API 또는 사람이 등록한 글만 표시한다.
- 제목, 출처, 날짜, 원문 링크, 태그, 관련 교재 글만 저장한다.
- 자동 수집 글과 검토 완료 학습 노트를 분리한다.
- `검토 필요`, `교재 반영 완료` 같은 상태를 표시한다.

자동화 우선순위는 낮습니다. 먼저 수동 큐레이션으로 품질 기준을 만든 뒤 RSS/API 수집을 붙입니다.

## 오픈소스 기여 보완

기여자는 거대한 글 하나를 새로 쓰지 않아도 참여할 수 있어야 합니다.

기여 단위:

- 어려운 문장 쉽게 바꾸기
- 공식 출처 링크 추가
- 출처 확인일 갱신
- 용어 사전 항목 추가
- 복습 질문 추가
- 깨진 외부 링크 수정
- 모바일 가독성 문제 수정

필요한 문서:

- `CONTRIBUTING.md`: 구현됨
- 글 작성 템플릿: 구현됨
- 출처 추가 템플릿: 구현됨
- 산업 업데이트 템플릿: 구현됨
- 용어 사전 추가 기준: 구현됨
- PR 체크리스트

## Phase 1.1 작업안

다음 단계는 기능 확장보다 글 상세 경험과 출처 신뢰를 먼저 보완합니다.

| 우선순위 | 작업 | 완료 기준 |
| --- | --- | --- |
| P0 | 글 상세 90초 요약 카드 | 구현됨: DRAM/HBM/EUV 3편 상단에 비유, 예상 소요 시간, 레벨별 읽기 목표까지 표시 |
| P0 | 핵심 구조 다이어그램 | 구현됨: DRAM/HBM/EUV 3편에 구조, 병목, trade-off 라벨형 도식 표시 |
| P0 | 핵심 글 3편 리라이트 | 구현됨: DRAM/HBM/EUV에 스펙 읽기, trade-off, 실무 판단 기준 보강 |
| P0 | 출처 확인일 표시 | 구현됨: 출처 확인일, 사용 근거, 공식 자료 해석 블록 표시 |
| P1 | `/sources` 링크 허브 | 10개 이상 공식 출처 카드와 외부 이동 |
| P1 | 글 상세 관련 용어 탐색 | 구현됨: 핵심 용어, 현업 키워드, 태그 기반 용어 카드와 `/glossary` 앵커 연결 |
| P1 | 용어 사전/복습 질문 확장 | 용어 30개, 검색/필터 UX, 복습 질문 20개 구현 |
| P1 | 기여 가이드 | 구현됨: 작은 문서 기여 단위와 템플릿 명시 |
| P2 | `/industry` 수동 큐레이션 | 1차 구현: 공식 글 8개를 교재와 연결 |
| P2 | `/industry/[slug]` 상세 해설 | 구현됨: 원문별 질문, 출처 정책, 관련 교재 연결 |
| P2 | RSS/API 수집 설계 구현 | 공식 feed가 있는 출처만 메타데이터 수집 |

## 설계 판단 기준

새 보완안은 아래 질문을 통과해야 합니다.

1. 초보자가 시작점을 더 빨리 찾는가?
2. 중급자가 구조와 비교를 더 잘 이해하는가?
3. 심화 학습자가 실무 제약을 더 잘 볼 수 있는가?
4. 공식 자료 원문으로 이동하는 길이 명확한가?
5. 개인정보 없이도 학습 흐름이 이어지는가?
6. 포트폴리오 검토자가 기획, 콘텐츠, 구현 판단을 확인할 수 있는가?

이 기준을 통과하지 못하는 기능은 Phase 1.1에 넣지 않습니다.
