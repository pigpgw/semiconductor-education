# 로그인 없는 제품 정책

## 원칙

Semiconductor Education은 로그인 없이 사용할 수 있어야 합니다. 반도체 교육 접근성을 높이는 것이 목적이므로, 첫 방문자가 계정 생성 단계에서 멈추면 안 됩니다.

## 포함하지 않는 기능

- 회원가입
- 소셜 로그인
- 이메일 인증
- 세션 기반 인증
- 개인 프로필
- 서버 저장 학습 기록
- 이름, 이메일, 학교, 지역 같은 개인정보 입력

## 허용하는 저장 방식

- 현재 브라우저의 `localStorage`
- URL query parameter
- 사용자가 직접 저장하는 Markdown/텍스트 학습 노트

현재 `/study`의 체크리스트·학습 노트는 `localStorage`에만 저장합니다. 서버 계정이나 쿠키 기반 인증을 사용하지 않습니다.

## 로컬 학습 관리

- `/study`는 진도 체크와 학습 노트 템플릿을 제공합니다.
- 데이터는 현재 브라우저에만 남습니다.
- 다른 기기 동기화는 제공하지 않습니다.
- 사용자는 노트를 복사해 개인 메모 앱에 저장할 수 있습니다.

## 이유

- 지방이나 오프라인 교육 접근이 어려운 학습자도 바로 시작할 수 있어야 합니다.
- 전연령 학습자에게 개인정보 입력 부담을 주지 않아야 합니다.
- 포트폴리오 관점에서도 콘텐츠 구조, 학습 UX, 구현 품질을 보여주는 것이 우선입니다.
- 계정 시스템은 보안, 개인정보, 운영 책임을 키우므로 MVP 목적과 맞지 않습니다.
