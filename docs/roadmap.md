# 학습 로드맵

이 로드맵은 “용어 암기 순서”가 아니라 “반도체를 실무 문맥으로 이해하는 순서”입니다. 각 단계는 기초, 중급, 심화 독자가 같은 길을 다른 깊이로 따라갈 수 있게 설계합니다.

## 1. 컴퓨터와 전기 신호

- 전압, 전류, 저항
- 디지털 신호와 아날로그 신호
- CPU, 메모리, 저장장치의 역할

기획 의도:

- 반도체가 갑자기 어려워지는 이유는 전기 신호와 컴퓨터 구조가 연결되지 않기 때문입니다.
- 첫 단계에서는 수식보다 역할과 흐름을 우선합니다.

## 2. 트랜지스터와 메모리 셀

- MOSFET의 스위치 역할
- DRAM의 1T1C 구조
- NAND 플래시의 전하 저장
- 휘발성과 비휘발성

기획 의도:

- 제품 이름을 외우기 전에 데이터가 물리적으로 어떻게 저장되는지 보여 줍니다.
- 같은 “메모리”라도 빠른 작업용과 오래 보관용이 갈라지는 이유를 설명합니다.

## 3. DRAM과 NAND

- DRAM refresh
- DDR, LPDDR, GDDR, HBM
- NAND와 SSD
- V-NAND와 3D 적층

기획 의도:

- 삼성전자와 SK하이닉스 자료를 읽을 때 가장 자주 만나는 제품군을 먼저 정리합니다.
- 용도별 메모리 차이를 성능, 전력, 용량, 비용으로 비교합니다.

## 4. HBM과 AI 메모리

- HBM의 적층 구조
- TSV와 넓은 I/O
- Memory wall
- CXL, PIM, HBF

기획 의도:

- AI 시대의 핵심 병목이 연산만이 아니라 데이터 이동이라는 점을 이해시킵니다.
- HBM을 단일 제품이 아니라 GPU, 패키징, 데이터센터 전력과 연결합니다.

## 5. 공정과 미세화

- 웨이퍼와 노광
- DUV와 EUV
- 멀티 패터닝과 싱글 패터닝
- 수율과 생산성

기획 의도:

- 미세화는 “작게 만들면 끝”이 아니라 공정 복잡도, defect, 비용, 생산성과 연결됩니다.
- EUV를 장비 소개가 아니라 DRAM scaling을 위한 기술 선택으로 설명합니다.

## 6. 패키징과 시스템 관점

- Advanced packaging
- MR-MUF와 열 방출
- GPU와 메모리 연결
- AI 인프라의 전력 효율

기획 의도:

- 성능은 칩 하나만으로 결정되지 않습니다.
- 패키징, 열, 전력, 고객 검증이 실무 판단에서 왜 중요한지 다룹니다.

## Phase별 공개 계획

| Phase | 목표 | 공개 단위 |
| --- | --- | --- |
| Phase 1 | MVP 신뢰 확보 | DRAM, HBM, EUV 핵심 글 3편과 레벨 진단 |
| Phase 1.1 | MVP 품질 보강 | 핵심 글 1차 리라이트, 출처 해석 고도화, 요약 카드 고도화 |
| Phase 1.2 | 학습 루프 강화 | 용어 사전 탐색 고도화, 산업 해설 노트, 레벨별 읽기 안내, 학습 노트 내보내기 |
| Phase 2 | 메모리 제품군 확장 | NAND, SSD, DDR/LPDDR/GDDR 비교 |
| Phase 3 | 공정/패키징 확장 | 8대 공정, 포토, 식각, 증착, TSV, MR-MUF |
| Phase 4 | 실무 관점 확장 | 직무별 읽기 경로, 면접 질문, 공식 자료 해석 훈련 |

현재 1차 MVP 구현 상태는 [MVP 정리](mvp-summary.md), 다음 보완 우선순위는 [보완 기획](planning-improvements.md)을 기준으로 합니다.

## 우선순위 기준

새 주제는 다음 기준으로 우선순위를 정합니다.

- Samsung/SK 공식 자료에서 자주 등장하는가
- 초보자가 막히는 선수지식인가
- AI 메모리, 공정 미세화, 패키징처럼 산업 중요도가 높은가
- 포트폴리오에서 기술 구조화 능력을 보여 주는가
- 한 글 안에 기초, 중급, 심화 내용을 모두 담을 수 있는가
