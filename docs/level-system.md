# 레벨 시스템

## 목표

누구나 쉽게 시작하지만 내용은 가볍지 않게 만듭니다. 레벨은 학습자를 나누기 위한 벽이 아니라, 같은 주제를 어느 깊이로 읽을지 정하는 안내입니다.

## 레벨 설계 원칙

- 레벨은 콘텐츠 접근 권한이 아니라 읽기 안내입니다.
- 사용자는 진단을 받거나 직접 선택할 수 있습니다.
- 낮은 레벨을 선택해도 심화 내용은 숨기지 않습니다. 대신 먼저 읽을 구간을 알려 줍니다.
- 높은 레벨을 선택해도 기초 설명을 건너뛰도록 강제하지 않습니다. 실무자는 쉬운 설명으로 자기 설명력을 점검할 수 있어야 합니다.

## Level 1: 기초

대상:

- 반도체를 처음 듣는 중고등학생, 비전공자, 전공 초입 학습자
- CPU, 메모리, 저장장치 차이가 아직 흐릿한 사람
- 수식보다 비유와 그림으로 먼저 잡아야 하는 사람

제공 방식:

- 한 줄 결론
- 쉬운 비유
- 최소 용어 설명
- 생활 예시
- 체크 질문

완료 기준:

- DRAM, NAND, HBM, EUV가 각각 어떤 문제를 푸는지 말할 수 있다.

## Level 2: 중급

대상:

- DRAM과 SSD 차이는 알지만 구조와 병목 설명이 부족한 사람
- HBM, TSV, refresh, EUV 같은 용어를 연결하고 싶은 사람
- 삼성전자와 SK하이닉스 기술 글을 자기 말로 풀고 싶은 사람

제공 방식:

- 구조도 관점 설명
- 표 기반 비교
- 성능, 전력, 비용, 용도 트레이드오프
- 공식 출처 문장 해석

완료 기준:

- 왜 특정 메모리나 공정이 특정 용도에 맞는지 설명할 수 있다.

## Level 3: 심화

대상:

- 채용 면접, 기술 블로그, 포트폴리오에서 깊이를 보여주고 싶은 사람
- 산업 키워드를 단순 정의가 아니라 의사결정 기준으로 이해하고 싶은 사람
- 공정, 패키징, 수율, 고객 검증, AI 인프라 병목까지 보고 싶은 사람

제공 방식:

- 공식 자료와 기술 블로그 교차 확인
- 병목과 제약 조건 분석
- 양산성, 수율, 열, 전력, 패키징 관점
- 현업 질문과 체크리스트

완료 기준:

- 기술 홍보 문구와 실제 엔지니어링 의미를 분리해 설명할 수 있다.

## 글 하나의 내부 깊이

각 글은 다음 층으로 구성합니다.

1. 초보자 결론: “그래서 이 기술은 무엇인가”
2. 직관: 비유와 생활 예시
3. 구조: 실제 소자, 공정, 패키징
4. 트레이드오프: 성능, 전력, 비용, 수율
5. 실무 관점: 병목, 양산성, 고객 검증, 로드맵
6. 출처: 공식 자료와 심화 참고 자료

## 추천 레벨 계산

진단 질문은 8개를 기준으로 확장합니다.

| 점수 | 추천 레벨 |
| --- | --- |
| 0-5점 | 기초 |
| 6-10점 | 중급 |
| 11-16점 | 심화 |

추천 결과에는 다음 안내를 붙입니다.

- 기초: 결론, 비유, 용어 박스, 체크 질문부터 읽으세요.
- 중급: 구조 설명, 비교표, 공식 자료 해석까지 읽으세요.
- 심화: 트레이드오프, 양산성, 고객 검증 질문까지 읽으세요.

## 막힘 방지 장치

- 용어가 막히면 `/glossary`에서 쉬운 정의와 실무 맥락을 함께 확인한다.
- 글 상세 상단에서 기초, 중급, 심화 독자가 어디를 중심으로 읽어야 하는지 안내한다.
- 학습자는 한 번에 완독하지 않아도 된다. 기초 설명만 읽고 돌아와도 다음 방문에서 구조와 실무 관점으로 올라갈 수 있어야 한다.
- `/practice`에서 읽은 내용을 말로 설명하고, 답변의 실무 포인트를 통해 자기 이해도를 확인한다.
