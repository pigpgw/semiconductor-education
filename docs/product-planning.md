# 제품 기획 보완

## 한 줄 정의

Semiconductor Education은 로그인 없이 바로 시작할 수 있는 한국어 반도체 학습 사이트입니다. 초보자는 비유와 그림으로 시작하고, 중급자는 구조와 비교로 이해하며, 심화 학습자는 실무 의사결정 기준까지 따라갑니다.

## 기획 문제

반도체 자료는 이미 많지만 학습자가 바로 따라가기 어렵습니다.

- 공식 기술 자료는 최신성과 신뢰성이 높지만, 초보자를 위한 배경 설명이 부족합니다.
- 온라인 강의는 접근성이 좋지만 강좌 단위라 길고, 특정 기업 기술 흐름과 연결하기 어렵습니다.
- 기술 블로그는 깊이가 있지만 선수지식이 없으면 용어에서 막힙니다.
- 오프라인 실습 교육은 장비와 지역 접근성의 영향을 크게 받습니다.

따라서 이 프로젝트의 역할은 “공식 자료를 복사하는 사이트”가 아니라 “공식 자료를 읽을 수 있는 힘을 길러 주는 해설형 교재”입니다.

## 핵심 포지셔닝

| 축 | 일반 기술 블로그 | 대학/강의형 콘텐츠 | 이 프로젝트 |
| --- | --- | --- | --- |
| 진입 난이도 | 글마다 다름 | 선수지식 부담이 큼 | 레벨 진단으로 시작 |
| 최신 산업 연결 | 강함 | 느릴 수 있음 | Samsung/SK 공식 자료 기반 |
| 실무 맥락 | 글쓴이 경험 의존 | 이론 중심일 수 있음 | 병목, 수율, 패키징, 고객 검증까지 연결 |
| 학습 지속성 | 다음 글 연결이 약함 | 강좌 단위 진행 | 로드맵, 체크 질문, 로컬 학습 기록 |
| 접근성 | 대체로 무료 | 일부 유료/장기 과정 | 로그인 없는 무료 읽기 중심 |

## 대상 사용자

### 1. 처음 시작하는 학습자

- 중고등학생, 비전공자, 전공 초입 학습자
- 목표: DRAM, NAND, HBM, EUV가 각각 무엇인지 말할 수 있다.
- 필요한 경험: 쉬운 비유, 짧은 문단, 용어 사전, 그림 중심 설명

### 2. 취업 준비생과 전공 학습자

- 반도체 직무, 삼성전자 DS, SK하이닉스, 장비/소재/부품 기업에 관심 있는 사람
- 목표: 제품과 공정의 차이를 구조적으로 설명하고 면접 질문에 대비한다.
- 필요한 경험: 비교표, 트레이드오프, 공식 자료 근거, 체크 질문

### 3. 지방/비수도권 학습자

- 오프라인 교육, 실습 장비, 멘토 접근이 어려운 사람
- 목표: 혼자서도 순서대로 읽고 자기 수준을 확인한다.
- 필요한 경험: 로그인 없는 진입, 자기 속도 학습, 선수지식 안내, 복습 루프

### 4. 심화 학습자와 포트폴리오 평가자

- 기술 깊이와 구조화 능력을 보고 싶은 사람
- 목표: 이 프로젝트가 단순 요약이 아니라 산업 자료를 재구성한 결과임을 확인한다.
- 필요한 경험: 출처 정책, 콘텐츠 기준, 로드맵, 실무 관점 질문

## 레벨 판단 설계

레벨은 사용자를 나누는 벽이 아니라, 같은 글을 어느 깊이로 읽을지 정하는 안내입니다.

| 레벨 | 사용자 상태 | 사이트가 해줄 일 | 완료 기준 |
| --- | --- | --- | --- |
| 기초 | 용어가 낯설고 컴퓨터 구조가 흐릿함 | 비유, 그림, 한 줄 결론으로 진입 | DRAM/HBM/EUV가 푸는 문제를 말한다 |
| 중급 | 용어는 들어봤지만 구조 연결이 약함 | 셀, TSV, 패터닝, 대역폭을 비교 | 왜 이 기술이 특정 용도에 맞는지 설명한다 |
| 심화 | 산업 자료와 면접 질문까지 연결하고 싶음 | 병목, 수율, 열, 전력, 양산성까지 분석 | 기술 홍보 문구와 엔지니어링 의미를 분리한다 |

## 첫 방문자 진단 질문

자가 진단은 정답 시험이 아니라 읽기 깊이를 추천하는 도구입니다. 각 질문은 `모른다`, `대충 안다`, `설명할 수 있다` 중 하나로 답하게 합니다.

1. CPU, 메모리, 저장장치의 역할 차이를 설명할 수 있는가?
2. DRAM과 NAND가 왜 다르게 쓰이는지 말할 수 있는가?
3. DRAM refresh가 왜 필요한지 설명할 수 있는가?
4. HBM이 일반 DRAM보다 AI에 유리한 이유를 bandwidth 관점에서 설명할 수 있는가?
5. TSV, 적층, 패키징이 HBM에서 왜 중요한지 연결할 수 있는가?
6. DUV와 EUV의 차이를 wavelength, patterning, 생산성 관점으로 설명할 수 있는가?
7. 성능이 좋아지는 기술이 왜 항상 쉽거나 싸게 양산되지 않는지 말할 수 있는가?
8. 삼성반도체나 SK하이닉스 공식 글을 읽고 핵심 주장과 근거를 분리할 수 있는가?

추천 기준:

- 총점 0-5점: 기초
- 총점 6-10점: 중급
- 총점 11-16점: 심화
- 단, 사용자는 언제든 직접 레벨을 바꿀 수 있어야 합니다.

## 콘텐츠 구조

모든 글은 하나의 주제를 세 개의 깊이로 다룹니다.

1. 초보자 결론: 그래서 이 기술이 무엇인가
2. 생활 비유: 왜 필요한지 감각적으로 이해
3. 실제 구조: 셀, 공정, 패키지, 시스템 연결
4. 비교와 트레이드오프: 성능, 전력, 비용, 수율, 용도
5. 현업 키워드: 양산성, 고객 검증, 로드맵, 공급 안정성
6. 공식 자료 읽기: Samsung/SK 자료에서 어떤 표현을 근거로 봐야 하는가
7. 체크 질문: 정의 암기가 아니라 자기 말 설명을 요구

## MVP 기획 범위

MVP는 많은 글보다 “깊이 있는 3편 + 레벨별 학습 경험”을 먼저 증명합니다.

| 화면 | 역할 | 성공 기준 |
| --- | --- | --- |
| `/` | 사이트 목적과 핵심 주제 제시 | 5초 안에 DRAM/HBM/EUV 학습 사이트임을 이해 |
| `/level` | 자기 수준 판단 | 처음 온 사용자가 읽을 시작점을 고름 |
| `/roadmap` | 전체 학습 지도 | 기초부터 실무까지 이어지는 길을 봄 |
| `/learn` | 글 선택 | 난이도와 주제로 필요한 글을 찾음 |
| `/learn/[slug]` | 본문 학습 | 쉬운 설명에서 실무 관점까지 한 글에서 이동 |
| `/glossary` | 용어 막힘 해결 | 글을 벗어나도 다시 돌아올 수 있음 |
| `/practice` | 설명 능력 확인 | 읽은 내용을 자기 말로 답함 |
| `/study` | 로그인 없는 지속 학습 | 현재 브라우저에만 체크와 노트를 남김 |

## 첫 공개 콘텐츠

### DRAM은 왜 빠른 작업 메모리인가

- 기초: 작업대 비유, 휘발성, CPU가 바로 쓰는 데이터
- 중급: 1T1C, refresh, DDR/LPDDR/GDDR/HBM 차이
- 심화: 대역폭, latency, 전력, 서버/모바일/AI 용도 차이

### HBM은 왜 AI 시대의 핵심 메모리인가

- 기초: 데이터가 너무 많아 통로를 넓혀야 하는 문제
- 중급: TSV, stack, wide I/O, memory wall
- 심화: 패키징, 열, 수율, 고객 검증, HBM3E/HBM4 로드맵

### EUV는 왜 DRAM 미세화에 중요한가

- 기초: 더 얇은 펜으로 더 촘촘히 그리는 비유
- 중급: DUV 193nm, EUV 13.5nm, multi-patterning, single-patterning
- 심화: 장비 복잡도, defect, 생산성, scaling 한계

## 운영 원칙

- 로그인, 회원가입, 댓글, 좋아요, 조회수는 MVP에서 제외합니다.
- 개인정보를 요구하지 않습니다.
- 공식 자료를 근거로 삼되 문장을 복사하지 않습니다.
- 제품 홍보 문구는 그대로 믿지 않고, 기술적으로 어떤 의미인지 풀어 씁니다.
- 각 글은 `기초`, `중급`, `심화` 독자가 각각 얻을 것이 있어야 공개합니다.
- 최신성이 중요한 글은 `updatedAt`과 출처 확인일을 명시합니다.

## 성공 지표

초기에는 서버 추적보다 산출물 품질을 봅니다.

- 핵심 글 3편이 모두 출처, 체크 질문, 레벨별 안내를 포함한다.
- 용어 사전 30개 이상을 기초 설명과 실무 맥락으로 제공한다.
- 복습 질문 20개 이상을 난이도별로 제공한다.
- 모바일 360px에서도 본문, 표, 목차, 다이어그램이 깨지지 않는다.
- README와 docs만 봐도 프로젝트 의도, 출처 정책, 콘텐츠 기준이 이해된다.
- 채용/포트폴리오 검토자가 “기술을 구조화하고 사용자 경험으로 옮겼다”고 판단할 수 있다.

## 확장 로드맵

### Phase 1: MVP 신뢰 확보

- DRAM, HBM, EUV 핵심 글 완성
- 레벨 진단, 용어 사전, 복습 질문, 로컬 학습 노트 제공
- README와 기획 문서 정리

### Phase 2: 메모리 전체 지도 확장

- NAND, SSD, V-NAND
- DDR, LPDDR, GDDR 비교
- CXL, PIM, SOCAMM, eSSD

### Phase 3: 공정과 패키징 심화

- 8대 공정 입문
- 포토리소그래피, 식각, 증착, 계측
- Advanced packaging, chiplet, MR-MUF, thermal

### Phase 4: 실무/직무 관점

- 메모리 설계, 공정, 소자, 패키징, 품질, 제품기술 직무별 읽기 경로
- 공식 보도자료를 기술 질문으로 바꾸는 훈련
- 면접 대비용 설명 체크리스트
