# 디자인 원칙

## 목표

이 사이트는 기술 교재이면서 포트폴리오입니다. 따라서 첫 화면은 관심을 끌어야 하지만, 장식보다 학습 이해를 우선합니다.

## 시각 설계

- 홈 첫 화면은 다크 문서형 레이아웃, 칩 로고, 기술 그리드, stats strip, 학습 트랙 카드로 구성한다.
- 첫 화면에서 “무료 반도체 학습 사이트”와 “기초부터 실무 관점까지”가 동시에 보여야 한다.
- 카드, 표, 용어 박스, 콜아웃은 모두 실제 학습 행동을 돕는 용도로만 사용한다.
- 과한 그라디언트, 떠다니는 장식, 마케팅식 히어로는 사용하지 않는다.
- 주제별 다이어그램은 글 상세나 카드에서 구조 이해를 돕는 경우에만 사용한다.

## 가독성 설계

- 본문 폭은 긴 줄이 되지 않게 제한한다.
- 문단 간격을 넉넉하게 두어 전연령 학습자가 모바일에서도 읽기 쉽게 한다.
- 표는 모바일에서 가로 스크롤을 허용해 내용이 찌그러지지 않게 한다.
- 기술 키워드는 배지와 용어 설명으로 분리해 본문 흐름을 끊지 않는다.

## 관심 유도 방식

- 첫 화면에서 커리큘럼, 핵심 글, 출처, 무료 접근 정책을 빠르게 파악하게 한다.
- 학습 트랙은 색상, 아이콘, 키워드 배지로 구분하되 한 가지 색에만 의존하지 않는다.
- 문서 프리뷰는 사용자가 실제 글의 읽기 방식을 미리 이해하게 만드는 장치로 둔다.
- 글 상세에는 본문 전에 핵심 구조와 레벨별 읽기 안내를 배치해 독자가 읽을 준비를 하게 한다.

## 최종 UI 구성

- sticky header: 칩 로고, 주요 경로, GitHub 링크
- hero: 한 줄 가치 제안, 레벨 진단, 교재 진입, 공식 출처 태그
- stats strip: 레벨, 핵심 글, 로드맵, 무료 학습 정보
- curriculum: 5개 학습 트랙 카드
- doc preview: 사이드바, 핵심 콜아웃, 코드형 읽기 흐름
- roadmap: 단계별 타임라인
- latest docs: 첫 공개 글 3편 진입
- CTA: 로그인 없이 바로 시작하는 다음 행동
