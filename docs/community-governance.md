# 커뮤니티 운영 기준

Semiconductor Education은 한 사람이 완성해서 닫는 프로젝트가 아니라, 여러 사람이 공식 자료를 함께 읽고 더 좋은 설명으로 다듬어 가는 공개 문서입니다. 이 문서는 기여자가 무엇을 어떻게 고치면 되는지, 리뷰어가 무엇을 기준으로 판단해야 하는지 정리합니다.

## 운영 원칙

1. 문서 품질이 기능보다 우선입니다.
2. 모든 기술 주장은 공식 출처와 연결합니다.
3. 쉬운 설명과 실무 관점을 함께 유지합니다.
4. 작은 기여를 환영합니다.
5. 원문 복제보다 원문으로 이동할 수 있는 링크와 학습 질문을 제공합니다.

## 기여 레벨

| 레벨 | 기여 예시 | 리뷰 기준 |
| --- | --- | --- |
| Level 1 | 오탈자, 문장 다듬기, 깨진 링크 수정 | 의미가 유지되고 가독성이 좋아졌는가 |
| Level 2 | 용어 사전, 복습 질문, 출처 확인일 보강 | 초보자 설명과 실무 맥락이 함께 있는가 |
| Level 3 | 기존 글의 구조/비교/trade-off 보강 | 공식 출처 근거와 판단 기준이 분명한가 |
| Level 4 | 새 교재 글, 새 학습 트랙, 새 산업 해설 | 로드맵, 콘텐츠 기준, 출처 정책과 맞는가 |

처음 기여하는 사람은 Level 1 또는 Level 2부터 시작하는 것을 권장합니다. 큰 글을 한 번에 쓰는 것보다 작은 PR을 여러 번 나누는 방식이 리뷰와 유지보수에 좋습니다.

## 콘텐츠 상태

새 글이나 큰 문서 수정은 아래 상태를 기준으로 관리합니다.

| 상태 | 의미 | 다음 행동 |
| --- | --- | --- |
| `proposal` | 주제와 출처 후보만 있는 상태 | 이슈로 필요성, 대상 독자, 공식 출처를 확인합니다. |
| `draft` | 초안은 있으나 검증이 부족한 상태 | 문장 구조, 레벨별 설명, 출처를 보강합니다. |
| `source-check` | 기술 주장과 공식 출처를 대조하는 상태 | `sources`, 확인일, 본문 근거를 점검합니다. |
| `review` | 공개 전 리뷰 상태 | 가독성, 모바일 UI, 검증 명령을 확인합니다. |
| `ready` | 공개 가능한 상태 | `dev`에 병합하고 다음 배포 후보로 둡니다. |

## 리뷰 기준

리뷰는 “맞다/틀리다”만 보는 절차가 아닙니다. 다음 네 가지를 함께 봅니다.

- 이해성: 처음 보는 사람이 첫 문단에서 주제를 잡을 수 있는가
- 깊이: 구조, trade-off, 수율, 전력, 열, 패키징, 고객 검증 중 필요한 제약을 다루는가
- 근거: 공식 자료 링크와 확인일이 있고, 원문 문장을 복사하지 않았는가
- 흐름: 기초 모드, 중급 모드, 고급 모드, 본문, 복습이 서로 연결되는가

## 이슈와 PR 사용 방식

- 아이디어는 먼저 이슈로 남깁니다.
- 작은 문장 수정이나 오탈자는 바로 PR을 열어도 됩니다.
- 새 글은 `lesson proposal` 이슈를 먼저 만들고, 출처 후보와 목차가 확인된 뒤 PR로 옮깁니다.
- PR은 한 가지 목적만 담습니다. 예를 들어 “HBM 글 문장 개선”과 “패키징 새 글 추가”는 분리합니다.
- 기술 주장 수정 PR은 공식 출처 링크와 확인일을 PR 설명에 적습니다.

## 의사결정 기준

의견이 갈리면 아래 순서로 판단합니다.

1. 공식 출처와 최신 확인일이 있는가
2. 초보자에게 더 쉽게 읽히는가
3. 실무 관점의 정확성을 해치지 않는가
4. 기존 로드맵과 콘텐츠 모델을 깨지 않는가
5. 유지보수자가 계속 관리할 수 있는 범위인가

새 기능은 문서 학습 흐름을 분명히 개선할 때만 추가합니다. 로그인, 댓글, 좋아요, 조회수, 원문 크롤링 저장처럼 문서 신뢰와 직접 관련이 낮은 기능은 우선순위를 낮춥니다.

## 릴리스 흐름

```txt
issue/proposal -> task branch -> PR -> dev -> release PR -> main
```

- `dev`는 다음 공개 후보입니다.
- `main`은 안정 공개 문서입니다.
- 배포 전에는 `npm run validate`, `npm run check:links`, `npm run check:viewport`를 확인합니다.
- 외부 공식 사이트가 일시적으로 403을 반환하는 경우, 허용 목록에 있는지 확인하고 결과를 PR에 남깁니다.
