# MVP 정리

이 문서는 현재 1차 MVP가 어디까지 완성됐는지 기록합니다. 앞으로 보완 작업을 할 때 “이미 완성된 범위”와 “다음에 늘릴 범위”를 섞지 않기 위한 기준입니다.

## MVP 한 줄 정의

Semiconductor Education MVP는 로그인 없이 바로 시작하는 반도체 오픈소스 문서입니다. 첫 공개 범위는 레벨 진단, 전체 로드맵, 핵심 글 3편, 용어 사전, 복습 질문, 공식 출처 허브, 로컬 학습 노트입니다.

## 현재 콘텐츠 인벤토리

| 항목 | 현재 수량 | 비고 |
| --- | ---: | --- |
| 핵심 글 | 3편 | DRAM, HBM, EUV 1차 리라이트 완료 |
| 레벨 | 3단계 | 기초, 중급, 심화 |
| 로드맵 단계 | 6단계 | 전기 신호부터 패키징/시스템 관점까지 |
| 용어 사전 | 30개 | 검색, 카테고리, 난이도 필터 제공 |
| 공식 출처 | 11개 | 메모리, 파운드리, 장비, 계측, 연구 공식 링크 |
| 산업 업데이트 | 8개 | 공식 링크, 학습 관점, 관련 교재 연결 |
| 산업 해설 노트 | 8개 | 원문 복제 없이 질문, 출처 정책, 관련 교재 표시 |
| 복습 질문 | 20개 | 기초 7개, 중급 7개, 심화 6개 |
| 실무 시나리오 | 3개 | 레벨별 1개 |
| 학습 체크리스트 | 7개 | localStorage 기반 |

## 첫 공개 핵심 글

| 글 | 핵심 목표 | 심화 포인트 |
| --- | --- | --- |
| DRAM은 왜 빠른 작업 메모리인가 | 작업 메모리, 휘발성, refresh 이해 | 대역폭, 지연 시간, 전력, 제품군 차이 |
| HBM은 왜 AI 시대의 핵심 메모리인가 | memory wall과 넓은 데이터 통로 이해 | TSV, 적층, 패키징, 열, 고객 검증 |
| EUV는 왜 DRAM 미세화에 중요한가 | 짧은 파장과 미세 패턴 이해 | multi-patterning, defect, 생산성, scaling 한계 |

핵심 글 3편은 1차 리라이트를 거쳐 스펙 읽기 순서, 시스템 병목, trade-off, 실무 판단 기준을 본문에 보강했습니다.

## MVP에서 증명하는 것

- 반도체 지식을 초보자 언어로 시작해 실무 키워드까지 올리는 구조화 능력
- Samsung/SK 공식 자료를 학습자 관점으로 재구성하는 출처 기반 글쓰기 방식
- 레벨 진단, 필터, MDX, 용어 사전, 복습 질문을 연결한 학습 UX
- 로그인 없이도 학습 흐름을 이어갈 수 있는 접근성 중심 설계
- 360px 모바일부터 데스크톱까지 깨지지 않는 문서형 반응형 UI

## MVP에서 제외한 것

- 로그인, 회원가입, 소셜 로그인
- 서버 기반 개인 진도 저장
- 댓글, 좋아요, 조회수, 커뮤니티 기능
- 많은 글을 얕게 늘리는 방식
- 비공식 자료에 의존한 최신 제품 주장
- 개인정보 수집 기반 분석

## 품질 기준

MVP는 다음 검증을 통과해야 합니다.

```bash
npm run validate
npm run check:links
npm run check:viewport
```

- README에서 프로젝트 목적, MVP 범위, 실행 방법, 협업 흐름을 확인할 수 있다.
- `docs/README.md`에서 문서 읽는 순서를 알 수 있다.
- `docs/final-design.md`에서 1차 MVP 설계 판단 기준을 확인할 수 있다.

- `docs/planning-improvements.md`에서 다음 보완 우선순위를 확인할 수 있다.

## 일단락 판단

현재 1차 MVP는 “서비스를 크게 확장하기 전, 포트폴리오로 설명 가능한 기준점”입니다. 다음 작업은 기능 추가보다 핵심 글 3편의 문장 다듬기, 복습 질문 필터, 공식 출처 업데이트 흐름 정리, NAND/SSD 확장을 우선합니다.
