# 홍보 패키지

이 문서는 Semiconductor Education v0.1.0을 공개한 직후 바로 사용할 홍보 문구와 채널별 메시지를 정리합니다. 과장된 성과보다 “함께 발전시키는 오픈소스 반도체 문서”라는 포지션을 분명하게 전달합니다.

## 핵심 포지션

```txt
Semiconductor Education은 반도체를 처음 배우는 사람도 DRAM, HBM, EUV를 실무 관점까지 이해할 수 있도록 만든 오픈소스 문서입니다.
```

## 한 줄 소개

- 기초부터 실무 관점까지 읽는 반도체 오픈소스 문서
- DRAM, HBM, EUV를 구조와 trade-off로 배우는 무료 문서
- 공식 기술 자료를 초보자 언어와 현업 관점으로 다시 설명하는 프로젝트

## 짧은 홍보문

```txt
반도체를 처음 배우는 사람도 DRAM, HBM, EUV를 실무 관점까지 이해할 수 있도록 오픈소스 문서를 만들고 있습니다.

공식 기술 자료를 그대로 옮기지 않고, 쉬운 비유 -> 실제 구조 -> trade-off -> 실무 질문 순서로 다시 설명합니다.

첫 MVP는 레벨 진단, 10단계 로드맵, 핵심 글 3편, 용어 사전, 복습 질문, 공식 출처 허브로 구성했습니다.
```

## 긴 홍보문

```txt
Semiconductor Education v0.1.0을 공개했습니다.

이 프로젝트는 반도체를 처음 배우는 사람도 DRAM, HBM, EUV 같은 핵심 키워드를 실무 관점까지 이해할 수 있도록 만든 오픈소스 문서입니다.

목표는 기능이 많은 교육 플랫폼이 아니라, Toss Frontend Fundamentals처럼 기준 있는 공개 기술 문서를 꾸준히 만들어 가는 것입니다.

첫 버전에는 다음을 담았습니다.

- 8개 질문 기반 레벨 진단
- 기초부터 실무 관점까지 이어지는 10단계 로드맵
- DRAM, HBM, EUV 핵심 글 3편
- 기초/중급/심화 학습 인터랙션
- 용어 사전과 복습 질문
- Samsung, SK hynix, Micron, TSMC, Intel, ASML 등 공식 출처 허브
- 로그인 없는 로컬 학습 노트

아직 완성된 교재가 아니라 함께 발전시키는 문서입니다. 오탈자, 어려운 설명, 공식 출처 보강, 새 글 제안을 GitHub Issue와 PR로 받을 수 있게 열어 두었습니다.
```

## 채널별 메시지

| 채널 | 초점 | 문구 방향 |
| --- | --- | --- |
| GitHub | 오픈소스, 기여, 문서 기준 | README, issue template, source policy 강조 |
| LinkedIn | 포트폴리오, 기술 문서화 | 어려운 기술을 구조화한 프로젝트로 소개 |
| Velog/Tistory | 제작기, 설계 과정 | 왜 로그인 없는 문서형 MVP로 만들었는지 설명 |
| 커리어리 | 취업 준비, 학습 경로 | 반도체 핵심 키워드를 레벨별로 공부하는 흐름 강조 |
| 디스콰이엇 | 제품/오픈소스 소개 | 무료 문서, 공식 출처, 커뮤니티 기여 강조 |

## GitHub Release 본문

```md
Semiconductor Education v0.1.0은 “로그인 없이 바로 읽는 반도체 오픈소스 문서”의 첫 공개 버전입니다.

### 포함된 것

- 8개 질문 기반 레벨 진단
- 10단계 반도체 학습 로드맵
- DRAM, HBM, EUV 핵심 글 3편
- 기초/중급/심화 학습 인터랙션
- 용어 사전 30개
- 복습 질문 20개와 실무 시나리오
- 공식 출처 허브와 산업 업데이트 해설 노트
- 로그인 없는 로컬 학습 체크리스트와 노트

### 다음 후보

- NAND/SSD 첫 교재 글
- DDR/LPDDR/GDDR/HBM 비교
- 패키징, TSV, MR-MUF 글
- 수율, 계측, 검사 입문
```

## 런칭 당일 순서

1. `docs/big10-source-curriculum` PR을 `dev`에 머지합니다.
2. `dev -> main` release PR을 만들고 CI를 통과시킵니다.
3. Vercel에서 `main` production 배포를 확인합니다.
4. `NEXT_PUBLIC_SITE_URL`을 실제 production URL로 설정합니다.
5. `/robots.txt`와 `/sitemap.xml`이 production URL을 바라보는지 확인합니다.
6. README 상단과 GitHub About Website에 production URL을 넣습니다.
7. `v0.1.0` Release를 생성합니다.
8. GitHub, LinkedIn, 블로그, 커리어리, 디스콰이엇 순서로 공유합니다.

## 홍보 후 확인할 것

- 사람들이 첫 화면에서 `내 레벨 찾기`를 누르는지
- README에서 production URL을 바로 찾을 수 있는지
- Issue로 오탈자와 설명 개선 제안이 들어오는지
- 공식 출처 링크 오류 제보가 있는지
- “어떤 글이 다음에 필요하다”는 피드백이 모이는지

- “현직자 수준 보장”
